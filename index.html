Ultra Short Term Wind Power Forecasting

Posted on 2018-08-20 | In projects

Problem Definition

To enhance the utilization of wind power and make wind power arrangement more convenient, we need a high-precise wind power forecasting tool. Moreover, due to the fact that it will take lots of unneccessary time and repeated work if we build different prediction tools for different wind farm, the tool is expected to be generalized.

In this project, our focus is set on ultra-short-term (within 4 hours) wind power prediction. The project objectives are:

  • Build a genralized prediction tool which can be applied to most kinds of wind farm.
  • The forecasting result is with high accuracy.
  • The tool will be integrated to the industrial platform GenPro developed by CyberInsight.

Design and Implementation

The wind power of an entire wind farm can be predicted as a whole, or it can be predicted as the sum of each turbine. Considering the interaction of neighbor turbines, we choose to take the entire wind farm as a whole predict target.

The simplest model is to use linear regression with transformation. For example, P ~ $v^3$, where P represents wind power and v represents wind speed. This method is with low accuracy, but easy to implement. So, we take it as a toy model, to figure out the relationship between wind power and given features.

The input dataset is with millions of data point and more than twenty features, including wind speed, wind direction, pressure, humidity, etc. If training with all features and such a large dataset, not only it will take so much time, but also overfitting is tend to happen. During this procedure, we have tried to use decision tree model to sifting highly correlated features. To accelerate the training efficiency, we tried to use the method of batch processing. Finally, we have choose to use the model of deep neural network with logistic regression and SVM to do the forecast.

The tool is wrapped with an HTML5 interface, and support real-time data prediction with visualized forecasting result.

Stochastic Circuits Optimization

Posted on 2018-08-10 | In projects

This is an research work, and here is the link to the paper:
Simultaneous Area and Delay Optimization for Stochastic Circuits by D Flip-flop Insertion

Problem definition

Stochastic Computing

This is an computing technique using digital circuits. The idea is to encode real values through the probabilities of ones on bit streams, and then performs arithmetic computation.

Bit Stream Representation of Real Values

For example, the bit stream 10011010 can represent 0.5, or 1/2 since there are 4 ones in the 8-bit-long bit stream.

Arithmetic Computation

AND gate can perform multiplication a·b while OR gate can perform a + b - a·b.
For example, given two bit streams, a = 11010111 represents 6/8 and b = 11001010 represents 4/8. Let them go through an AND gate bit by bit, the output will be c = 11000010 represents 3/8, where c = a·b.

Challenges

We want to reduce the computation time and large hardware overhead to generate independent stochastic bit streams.

It seems that this can be addressed by sharing the stochastic number generator (SNG). However, this will lead to the maximal correlation of the generated stochastic bit streams. If so, consider the example of AND gate above, a = 11010111 and b = 11000110, still represents 6/8 and 4/8. However, the output will become c = 11000110 represents 4/8, or 1/2. In this situation, the AND gate cannot perform multiplication any longer, but give out the result of min{a, b}.

Idea

To ensure that the stochastic circuit can give out the expected result, the input stochastic bit streams have to be decorrelated. We can fulfill this target through insertion of D-Flip-Flops (DFFs).

The insertion of DFFs can potentially break a critical path, and hence, reduce the circuit delay.

Further, we hope that under the premise of functional correctness, least number of DFFs should be inserted to reduce area overhead.

Therefore, the point is how to ensure that with least number of DFFs, the circuit still has an equvalent functionality. For a given circuit, we list a set of linear equalities and inequalities that guarantees the functional equivalence, based on the method to obtain the stochastic function of a stochastic circuit with DFF inserted. And the solution is calculated with the help of integer linear programming (ILP).

Mass Shooting Predictable Pattern

Posted on 2018-04-28 | In projects

Problem Definition

This is an analysis concentrated on cases of firearms-related accidental deaths in the United States. The target is to find if there is a predictable pattern of mass shootings.

The data used in this project is collected from Gun Violence Archive.

Statistical Analysis

We can look at how many accidental deaths happen each day. If accidental deaths happened as random events, # of accidental deaths/day would follow a poisson distribution.

Num per Day 0 1 2 3 4 5
Observed freq 109 133 77 29 16 1

Using the method of maximum likelihood, we can prove that the parameter $k$ of the assumed poisson distribution is the sample mean. Then, take the method of Null Hypothesis Pearson’s Test, the poisson distribtution parameter k can be calculated to be 1.21, with a 95% level of significance.

Besides, relation between acidental deaths and weekdays or monthes can be easily proved as following a uniform distribution.

Network Packet Trace Analysis

Posted on 2017-05-16 | In projects

This project focuses on analysis of the computer network procedure, especially for the TCP/IP protocal stack. Here, I use the tool of Wireshark to capture traces of a network packet.

Here are some selected conclusion:

  • Layer encryption:
    We should add a secure layer below the layer which will be added encryption. For example, if we want to add encryption to the TCP layer, we should add a secure layer between TCP and IP layers. Also, we can encrypt the highest layer. For example, add SSL between the HTTP and TCP layers. Then, every time for uploading, the higher layer message should pass down to the lower layer, and then the lower layer should append headers to the message. Then comes the secure layer add encryption to the appended headers. For downloading, the message will first pass through the decryption layer. Then, the decrypted message will pass through the lower layer normally, and after that, the higher layer.

  • Layer compression:
    Just like adding encryption, adding compression to a layer should add a compression layer below it. For example, the presentation layer in the OSI-model, adds compression to the application layer. We can add one compression layer to the layer which we want to add compression to. Since there are boundaries between a lower-layer header and the message from the upper layer, the new layer can set a rule to only do compression for the header part. And when downloading, the new layer do de-compression for the header part again. The normal de-compressed message will then be passed to the lower layer, then the higher one.

  • TCP reliability:
    There are several kinds of conditions which may be raised by a segement loss. The first one is duplicated ACK. Dup ACK occurs because when segements are lost, the receiver may receive some frames with a sequence number larger than expected, and it’ll then send an ACK with the expected sequence number, causing duplicated ACK. Theoratically, 3 continuous Dup ACKs will cause a retransmission. When Wi-Fi connection is cut down, a trace of such can be captured, and when the Wi-Fi connection was rebuilt, TCP triggers a fast retransmission. Therefore, it still can be said that Dup ACKs will trigger retransmission.
    Notice that here TCP triggers a fast retransmission, not a ordinary transmission. This is because transmission is sent out when RTO is expired, (the sender doesn’t receive the corresponding ACK within the RTO), but when series of Dup ACKs show up, the sender will regard that frame as a lost one, though the RTO is still not expired.
    The Wi-Fi connection’s rebuildment can be proved by the analysis information in the next frame: it contains a new RTT estimation, (those frames sent out when the connection isbreakdown don’t contain an updated RTT value, but only the initial RTT value).
    The second problem may come up is TCP out-of-order, which will happen when the last received frame has a sequence number smaller than that of a segment received before. TCP may also do retransmission because of expired RTO. RTO is calculated by RTT. Since RTT changes a lot, RTO changes with it.

  • Concurrent TCP connection:
    When concurrent connections are built up, the client will handle two connections together, which is based on SSL. SSL helps listen to all these “threads”, to assign network resources and do management. In addition, SSL helps encrypt application data and exchange connection key for client and server. Also, to keep all the connections up, there is another interesting mechanism called TCP keep-alive. With keep-alive, TCP connection can be kept up when there is no communication between the specific client and server, (for example, when the client is communicating with another server), it’ll reduce the times to set up or release a connection, and enhance the efficiency. When several connections are set up, there’ll unavoidably be collisions and packet loss. When a segment loss is found, TCP will start retransmission / fast retransmission.

Model MIPS Computer

Posted on 2016-12-20 | In projects

Problem definition

The target is to model both single-cycle and pipelined implementation of MIPS computer in Verilog, which support MIPS instructions, such as *add*, *and*, *slt*, *lw*, *bne*, etc.

Besides, forwarding should be implemented in the pipelined structure to handle data and control hazards.

The processor will be synthesized by using Xilinx synthesis tool, and run on an FPGA board.

Design and Implementation

Due to the fact that different components fulfill different functions, the work can then be divided into two parts: design of components and design of integration.

The following functional components can be extracted from the architecture:

Instruction Fetch stage (IF):

  • PC stores instruction addresses.
  • Instruction memory stores machine codes.

Instruction Decode stage (ID):

  • Register stores the numbers.
  • Sign-extension sign-extends immediate numbers.
  • Control module generates all control signals.
  • Hazard detection detects possible data/control hazard.

Execute stage (EX):

  • ALU calculates the results based on inputs.
  • ALU Control generates ALU operation signals.
  • Forwarding generates forwarding control signals.

Memory Stage (MEM):

  • Data memory stores all data.

After that, these components are integreted with the pipeline registers.
